## What Is This?

One project a day. Every day. For a full year.

Not tutorials. Not courses. Actual hands-on builds — scripts, infra, pipelines, tools.
Each project takes 30–60 mins. The goal is consistency, not perfection.

## Ground Rules

- **30–60 mins per project.** If it takes longer, scope it down.
- **Every project gets a commit.** No project exists without a push to GitHub.
- **Every project gets a README.** What it does, how to run it, what you learned.
- **Folder structure:** `/day-001/`, `/day-002/` … each with code + README.
- **Missed a day?** Do it the next day. Streaks matter less than finishing.
- **These are personal learning projects.** Build for yourself, not for show.
